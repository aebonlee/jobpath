# 18. 회원 목록 jobpath 전용 필터링 + 이름 표시

## 작업일: 2026-04-14

## 문제

1. **회원 목록에 모든 auth.users 표시**: `forjob_profiles` 백필 시 공유 Supabase의 auth.users 전체(346명)가 삽입됨. www 사용자까지 jobpath 관리자 대시보드에 표시되는 문제
2. **이름 미표시**: 오늘 결제 완료한 9명의 주문자명(봉은수, 이미라, 이미숙 등)이 관리자 회원 목록에서 이메일만 보이고 이름이 표시되지 않음

## 원인 분석

- `forjob_profiles` 백필 SQL이 `auth.users` 전체를 대상으로 INSERT하여 www 전용 사용자까지 포함
- 회원 목록 빌드 로직이 `forjob_profiles`를 1차 소스로 사용 → 346명 전부 표시
- 회원 테이블에 "이름" 컬럼 자체가 없었음

## 해결 방법

### 1. 프론트엔드 — AdminDashboard.tsx 회원 목록 로직 변경

**변경 전**: profiles + orders + redemptions 3개 소스에서 이메일 통합
```typescript
const emailMap = {};
profiles.forEach(p => emailMap[p.email] = { source: '가입' });
orders.forEach(o => emailMap[o.email] = { source: '주문' });
redemptions.forEach(r => emailMap[r.email] = { source: '쿠폰' });
```

**변경 후**: orders + redemptions만 회원 소스, profiles는 이름 조회용만
```typescript
// 프로필 → 이름 조회 맵
const profileNameMap = {};
profiles.forEach(p => { if (p.name) profileNameMap[p.email] = p.name; });

// 주문 + 쿠폰 활동이 있는 회원만 포함
const emailMap = {};
orders.forEach(o => {
  emailMap[o.email] = { name: o.user_name || profileNameMap[o.email], source: '주문' };
});
redemptions.forEach(r => {
  if (!emailMap[r.email]) emailMap[r.email] = { name: profileNameMap[r.email], source: '쿠폰' };
});
```

### 2. 회원 테이블 UI — 이름 컬럼 추가

- `<th>이름</th>` 헤더 추가
- `<td>{member.name || '-'}</td>` 데이터 셀 추가
- 이름은 `forjob_orders.user_name` 우선, 없으면 `forjob_profiles.name` 사용

### 3. DB 정리 — 불필요한 프로필 삭제 SQL

```sql
DELETE FROM forjob_profiles
WHERE id NOT IN (SELECT DISTINCT user_id FROM forjob_orders WHERE user_id IS NOT NULL)
AND email NOT IN (SELECT DISTINCT user_email FROM forjob_orders WHERE user_email IS NOT NULL)
AND email NOT IN (SELECT DISTINCT user_email FROM forjob_coupon_redemptions WHERE user_email IS NOT NULL);
```
