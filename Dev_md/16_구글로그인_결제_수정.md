## 문제 상황
1. Google 로그인/가입 시 에러 발생

## 수정 내용

### 1. LoginPage.tsx - OAuth 에러 감지 추가
- URL에서 `error` / `error_description` 파라미터 감지
- 사용자에게 구체적인 에러 메시지 표시
- 에러 표시 후 URL 정리 (history.replaceState)

### 2. portone.js - SDK 로드 대기 로직 추가
- `async` 로딩된 SDK가 준비될 때까지 대기 (최대 10초)
- SDK 미로드 시 사용자 친화적 에러 메시지
- Demo mode 분리

## Supabase 대시보드 필수 설정

### Google OAuth 활성화
1. Supabase Dashboard > Authentication > Providers > Google
2. **Enable Sign in with Google** 활성화
3. Google Cloud Console에서 OAuth 2.0 Client ID/Secret 발급:
   - Google Cloud Console > APIs & Services > Credentials
   - OAuth 2.0 Client ID 생성
   - Authorized redirect URI: `https://hcmgdztsgjvzcyxyayaj.supabase.co/auth/v1/callback`
4. Client ID와 Client Secret을 Supabase에 입력

### Redirect URL 설정
1. Supabase Dashboard > Authentication > URL Configuration
2. **Site URL**: `https://jobpath.dreamitbiz.com`
3. **Redirect URLs**에 추가:
   - `https://jobpath.dreamitbiz.com`
   - `https://jobpath.dreamitbiz.com/**`
   - `https://www.dreamitbiz.com`
   - `https://www.dreamitbiz.com/**`
   - `http://localhost:5190` (개발용)

### PortOne 결제 설정
1. PortOne 관리자 > 결제대행사 설정
2. KG이니시스 (html5_inicis) 연동 확인
3. 가맹점 식별코드: `imp61949262`
4. 허용 도메인에 `jobpath.dreamitbiz.com` 추가
