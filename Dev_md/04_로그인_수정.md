## 원인 분석

### 1. 존재하지 않는 Supabase 프로젝트 URL

| 항목 | 값 | 상태 |
|------|---|------|
| 기존 .env URL | `qojjmfbtsyzrmhhkxpbf.supabase.co` | **DNS 해석 불가 (존재하지 않음)** |
| 공용 인스턴스 | `hcmgdztsgjvzcyxyayaj.supabase.co` | 정상 (competency, career 등 사용 중) |

- `ping` 테스트: 기존 URL은 "호스트를 찾을 수 없습니다" 오류
- 공용 인스턴스는 IP 정상 해석

### 2. Supabase 클라이언트 auth 설정 누락

기존 코드:
```js
export const supabase = createClient(supabaseUrl, supabaseAnonKey);
// → auth 옵션 없음! OAuth 콜백 처리 불가
```

다른 정상 동작 프로젝트(competency, career, allthat) 분석 결과:
- `detectSessionInUrl: true` 필수 (OAuth 콜백 감지)
- `flowType: 'pkce'` 권장 (보안 강화)
- `INITIAL_SESSION` 이벤트 기반 세션 처리

## 수정 사항

### 1. `.env` - Supabase URL 변경

```
VITE_SUPABASE_URL=https://hcmgdztsgjvzcyxyayaj.supabase.co
VITE_SUPABASE_ANON_KEY=(공용 인스턴스 키)
VITE_SITE_URL=https://forjob.dreamitbiz.com
```

- `forjob_` 테이블 접두어 사용으로 다른 프로젝트와 충돌 없음

### 2. `src/lib/supabase.js` - auth 옵션 추가

```js
export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  auth: {
    flowType: 'pkce',
    detectSessionInUrl: true,
    autoRefreshToken: true,
    persistSession: true,
  },
});
```

### 3. `src/contexts/AuthContext.jsx` - 세션 처리 개선

- `getSession()` 기반 → `INITIAL_SESSION` 이벤트 기반으로 변경
- 5초 fallback 타임아웃 추가 (무한 로딩 방지)
- `redirectTo`를 `VITE_SITE_URL` 환경변수로 관리
